# JeTTY v0.22.0

**Inline images, the Kitty way.** JeTTY now speaks the **Kitty graphics protocol**,
so image tools that target Kitty render a real GPU-textured bitmap right in the grid —
alongside the Sixel support from v0.19. Designed from a verified blueprint,
stress-tested by two design critics (4 blocking issues), then adversarially reviewed
at the code level across three lenses (0 blocking) and screenshot-verified.

## What it does

Programs transmit an image over an APC escape (`ESC _ G … ESC \`); JeTTY decodes it
and draws it at the cursor, reserving its cell rows so it scrolls with your
scrollback — in both the main and detached windows. Supported:

- **Transmit + display** (`a=t` / `a=T`), **put** a previously-transmitted image by
  id (`a=p`), **query** (`a=q`), and **delete** (`a=d`).
- Formats: **RGB** (`f=24`), **RGBA** (`f=32`), **PNG** (`f=100`).
- **Chunked** transfers (`m=1`) and **zlib compression** (`o=z`).

## What renders

| Tool | Out of the box? |
|---|---|
| `chafa -f kitty <img>` | ✅ streams base64 |
| `timg -p kitty <img>` | ✅ streams base64 |
| `kitten icat --transfer-mode=stream <img>` | ✅ direct base64 (+ compression, chunking) |
| `kitten icat <img>` (default) | ❌ needs `--transfer-mode=stream` |

`kitten icat`'s default local transfer uses a temp file / shared memory
(`t=f`/`t=t`/`t=s`), which JeTTY **refuses on purpose** — an untrusted terminal
stream must not open files or shared memory on your machine. Pass
`--transfer-mode=stream` and it renders.

## Safety & scope

Every decoder — the hand-rolled base64, RGB/RGBA, PNG, and zlib inflate — is bounded
*before* it allocates (dimensions capped at 4096×4096 / 16 Mpx, input and output byte
caps, a chunk-count cap) and never panics on hostile input, the same bar as the Sixel
decoder. **No new external dependency:** `png` and `miniz_oxide` were already in the
build. There's zero cost when no image is present — one cold scanner branch, the
render hot loop untouched.

Documented MVP non-goals (refused cleanly, never a crash): file / temp / shared-memory
transfer, animation, Unicode placeholders, server-side scale-to-cell-box, 8-bit APC.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
