# JeTTY v0.20.0

**Configurable keybindings.** Every shortcut in JeTTY is now yours to remap.
Designed from a verified blueprint, stress-tested by two design critics (6
blocking issues, most of them about *not* changing today's behavior for people
who don't touch the config), then adversarially reviewed.

## Make the keys yours

Drop a `[keys]` table in `~/.config/jetty/config.toml`:

```toml
[keys]
new_tab   = "Ctrl+T"                        # frees Ctrl+Shift+T back to the shell
paste     = ["Ctrl+Shift+V", "Shift+Insert"]  # more than one accelerator, if you like
font_up   = "Ctrl+Plus"
select_all = ""                             # "" = explicitly unbound
```

- **Anything you don't set keeps its exact current default** — a keymap oracle
  test proves the defaults are byte-for-byte identical to before, so upgrading
  changes nothing until *you* change something.
- It **hot-reloads live** — edit the config while JeTTY is running and the new
  bindings apply without a restart.
- Layout-independent bindings stay independent (font size still works on
  Turkish-Q / German-QWERTZ, not just US).

## Safe by design

You can't lock yourself out: a binding that would swallow a terminal **control
byte** (`Ctrl+C` SIGINT, `Ctrl+D` EOF, …) or a bare printable key is rejected with
a warning, and conflicting bindings are caught when the config loads. On macOS the
`Cmd` shortcuts fold in cleanly and unknown `Cmd` combos are still swallowed (not
typed into your shell).

Remappable actions: new / close / next / prev / select tab, detach, copy, paste,
select all, search, command palette, jump to prev/next prompt, open Settings,
font size up/down/reset, opacity up/down, hide, quit.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
