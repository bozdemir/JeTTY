# JeTTY v0.16.0

**SSH-ready & yours** — three things a daily-driver terminal needs, so nothing
sends you back to your old one. Designed from a verified blueprint, stress-tested
by two design critics (9 blocking issues caught before a line was written), then
adversarially reviewed.

## What's new

- 📋 **OSC 52 clipboard.** Yank in `nvim`, copy in `tmux`, select over `ssh` — it
  lands in your **local** system clipboard. Write works out of the box; **remote
  paste is opt-in** (`osc52_allow_paste = true`), because a terminal that lets any
  remote host read your clipboard is a terminal that leaks your passwords. Copies
  are capped at 100 KiB.

- 🎨 **Your own themes.** Drop a palette in `~/.config/jetty/themes/*.toml` — 16
  ANSI colors plus fg / bg / cursor / selection — and it shows up in the picker
  (Settings → Look). Name it after a built-in to **shadow** it. A broken theme
  file is skipped and logged, never fatal. The 22 built-ins are still there.

- ♻️ **Config hot-reload.** Edit `~/.config/jetty/config.toml` — or a theme file —
  and JeTTY applies it **live**: theme, opacity, font, effects, scrollback… no
  restart. It follows a **symlinked** config (dotfiles / stow / chezmoi), never
  fights its own Settings-panel writes, and — importantly — **never rewrites or
  resets your file** if a save is mid-flight or malformed; it just waits for the
  next good one. (`summon_hotkey` and `launch_at_login` still apply on restart.)

The file watcher is OS-event-driven (inotify / FSEvents), so idle CPU stays at
zero. One dependency added — `notify` — with no GTK/glib in its tree.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
