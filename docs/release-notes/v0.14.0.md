# JeTTY v0.14.0

**Shell integration (OSC 133).** JeTTY now understands where your prompts and
commands begin and end, so it can flag failures and let you jump between prompts.
Designed from a verified blueprint, stress-tested by two design critics, then
adversarially reviewed — with the prompt-mark model held to a strict
**"correct or absent, never wrong"** bar (the exit code is verified against a
real `zsh` + `powerlevel10k`).

## What you get

- 🔴 **Failed-command marker** — a themed bar on the prompt row of any command
  that exited non-zero. Scroll back and your failures stand out.
- 🧭 **Prompt jump** — `Ctrl+Shift+Z` (previous) / `Ctrl+Shift+X` (next) scroll
  straight to a shell prompt.
- 🧩 **`jetty --print-shell-integration <zsh|bash|fish>`** — opt in from your rc
  file; JeTTY **never touches your dotfiles**.

## Turn it on

```sh
# ~/.zshrc
[[ -n "$JETTY" ]] && command -v jetty >/dev/null 2>&1 && source <(jetty --print-shell-integration zsh) 2>/dev/null
```

**powerlevel10k users:** add `POWERLEVEL9K_TERM_SHELL_INTEGRATION=true` to your
`~/.p10k.zsh` — p10k emits the marks itself (instant-prompt-safe, correct exit
codes) and JeTTY reads them. (bash → `~/.bashrc`, fish → `~/.config/fish/config.fish`,
same shape.)

## Honest about the edges

Prompt marks are exact for the common case (default 10k scrollback, no resize
since the mark was made). By design they **degrade to absent — never wrong**:
after a window/font resize the grid rewraps, so marks are cleared and re-made on
the next prompt; and once the scrollback fills to its cap, marks pause until
history drops below it. JeTTY will never draw a marker on the wrong row. Inside
`tmux`/`screen`, OSC 133 reaches the multiplexer unless passthrough is set up.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
