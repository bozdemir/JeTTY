# JeTTY v0.19.0

**Inline images.** The biggest feature yet: JeTTY now renders real bitmaps in the
terminal via the **Sixel** protocol. Planned from a verified blueprint,
stress-tested by *three* design critics (9 blocking issues, most about decoding
untrusted image data without a single panic or OOM), then adversarially reviewed
and screenshot-verified.

## See images in your terminal

Any program that speaks Sixel now draws a real, GPU-textured bitmap right in the
grid — at native pixel size, on its own line, scrolling with your scrollback:

```bash
img2sixel photo.png          # a photo, inline
chafa -f sixel image.png     # or via chafa
python -c "...; plt.show()"  # a matplotlib plot, with a sixel backend
lsix *.png                   # thumbnail a folder
```

It works in **detached windows** too, and disappears cleanly when scrolled out of
your scrollback.

## Built to be safe

Decoding untrusted image bytes off the PTY is the dangerous part, so it got the
most scrutiny: the decoder is hand-rolled and hardened — bounded allocation,
bounds-checked pixel writes, saturating counts, size caps — so a malformed or
hostile Sixel can never panic or OOM the terminal. Image memory is capped with
LRU eviction, and there's **zero idle cost** when nothing is on screen. Colors
(RGB and DEC HLS) are correct, and no new dependency was added.

**Scope:** Sixel, static images. Kitty graphics / iTerm2 protocols and animation
are planned next — the interception, placement, and GPU image layer are built to
extend to them.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
