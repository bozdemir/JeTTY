# JeTTY v0.21.0

**No-mouse text extraction.** Two keyboard-first ways to grab what's on screen —
because reaching for the mouse to copy an error line breaks your flow. Designed
from a verified blueprint, stress-tested by two design critics, adversarially
reviewed, and screenshot-verified.

## Hint mode — `Ctrl+Shift+H`

Press it and JeTTY drops a short home-row label onto every **URL, file path, git
hash, and IP** on screen. Type the label to **copy** that token to your
clipboard; hold **Alt** to **open** a URL in your browser instead. Start typing a
label and the set narrows; `Esc` cancels. Tokens that wrap across the top or
bottom edge are captured whole.

## Copy-mode — `Ctrl+Shift+Space`

A vi-style keyboard cursor over the viewport and scrollback — no mouse:

- move with `h j k l`, `w b e` (word), `0` / `$`, `g` / `G`, `Ctrl+u` / `Ctrl+d`
- `v` starts a character selection, `V` a line selection
- `y` (or `Enter`) yanks the selection to the clipboard and exits; `Esc` exits

The selection is exact (inclusive on both ends), and the real cursor hides so you
only see the copy-mode one.

Both modes are **fully remappable** in `[keys]` (default `hint_mode` /
`copy_mode`), capture all input while active — nothing leaks to the shell, even
with an IME — and add zero cost when you're not using them.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
