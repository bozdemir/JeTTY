# JeTTY v0.15.0

**Run & Notify** — the feature a summon terminal is uniquely built for. JeTTY
spends its life hidden; now it tells you when your work is done. Designed from a
verified blueprint, stress-tested by two design critics, then adversarially
reviewed.

## The one you'll feel

Kick off a long build, hit F9 to send JeTTY away, go do something else — and the
moment the command finishes, JeTTY **pings you**:

- a native desktop notification naming the **tab**, the **exit code**, the
  **duration**, and the command's **last output line**;
- a taskbar / dock **urgency** hint;
- optional **auto-summon** — bring the window back (to the right tab) on finish.

It only fires when you're **not already watching** (hidden or unfocused), and
only for commands that ran past a threshold — so quick commands and anything you
can already see stay silent.

Turn the knobs in **Settings → Shell**: `notify_on_command_finish` (on),
`notify_min_seconds` (10), `notify_only_on_failure` (off), `auto_summon_on_finish`
(off).

Built on v0.14's OSC 133 shell integration, so it costs nothing at idle (it fires
only on the completion event) and the notification runs off the UI thread — a slow
or missing notification daemon can never stall the terminal. It's
desktop-environment-independent: any freedesktop daemon (KDE, GNOME, dunst, mako…)
works.

**Notes:** plain bash (no `bash-preexec`) has no command duration, so it's
failure-only; zsh / fish / powerlevel10k notify on success too. macOS gets the
dock-bounce urgency (rich toasts need an app bundle, future).

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
