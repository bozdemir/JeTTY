# JeTTY v0.23.1

**UI-font polish.** A patch release with four fixes so the chrome looks right
under any UI font — size or family — not just the default. Every change was
rendered and screenshot-verified.

## Fixes

- **Settings window auto-sizes to the UI font.** The Settings panel scales with
  the UI font, but the window was a fixed size, so a larger or wider UI font
  pushed the bottom rows (the UI-font dropdown, theme chips) off-screen. The
  window now derives its size from the scaled panel and re-fits on every font
  change — and it's user-resizable now too.
- **The neofetch welcome logo stays aligned.** The block-art "JETTY" wordmark is
  monospace ASCII that a proportional UI font turned into unreadable, skewed
  blocks. The whole welcome splash now renders in the terminal (monospace) font —
  which is what neofetch-style output actually is — so it's crisp in any UI font.
- **The perf HUD right-aligns correctly.** The bottom
  `⚡ ms · fps · CPU% · MB/s` strip computed its width as `chars × cell` (a
  monospace assumption), so under a proportional UI font it floated well left of
  the edge. It now measures its true rendered width and hugs the right edge at
  any font.
- **Redesigned the Keyboard Shortcuts (`?`) overlay** into grouped sections with
  aligned key / description columns and a colour hierarchy (header › key ›
  description), so it scans at a glance instead of being a dense grey wall.
  Remapped chords still show.

Full detail in the
[changelog](https://github.com/bozdemir/JeTTY/blob/main/CHANGELOG.md).

## Install

```bash
curl -fsSL https://raw.githubusercontent.com/bozdemir/JeTTY/main/install.sh | sh
```

Or a `.deb` / **AppImage** from the assets below (the installer verifies
`SHA256SUMS.txt`). Build from source: `cargo build --release`.
